Transaction Access Point (TAP) protocol interfaces 1.0

The “Transaction Access Point” (TAP) specification defines a standard protocol for RESTful exchange of e-business documents between ledger systems.

[ Base url: / ]

Post TAP message

/{tapUrl}

POSTPost TAP message to given TAP URL

Post TAP message. Payload must be a TAP message with a signature, each in a separate formData parameter. TAP URL could be any valid URL.

Get TAP message delivery status

/{tapUrl}/{messageId}/status

GETGet TAP message status

Doesn’t require any auth (knowing message ID is enough for access control). May be used for debug purposes, to see how messages are processed and possible errors. This is TAP protocol, TAP-GW just defines the URL format here.
